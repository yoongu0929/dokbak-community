# 요구사항 문서

## 소개

"독박육아 커뮤니티"는 혼자 육아를 담당하는 부모들을 위한 반응형 웹 애플리케이션입니다. 사용자는 로그인 후 대시보드를 통해 커뮤니티 활동을 확인하고, 게시판에서 정보를 공유하며, 월별 육아 꿀팁 랭킹 시스템을 통해 리워드를 받을 수 있습니다.

## 용어 정의

- **웹앱**: 독박육아 커뮤니티 반응형 웹 애플리케이션 전체 시스템
- **인증_모듈**: 사용자 로그인, 로그아웃, 세션 관리를 담당하는 모듈
- **대시보드**: 로그인 후 사용자에게 표시되는 메인 화면으로, 주요 활동 요약 정보를 제공하는 화면
- **게시판**: 사용자가 글을 작성, 조회, 수정, 삭제할 수 있는 커뮤니티 게시판
- **게시글**: 게시판에 작성되는 개별 글 단위
- **꿀팁_게시글**: 월별 육아 꿀팁 이벤트에 참여하기 위해 작성된 게시글
- **좋아요**: 사용자가 게시글에 대해 긍정적 반응을 표시하는 기능
- **랭킹_시스템**: 월별 꿀팁 게시글의 좋아요 수를 기반으로 순위를 산정하는 시스템
- **리워드**: 월별 랭킹 상위 사용자에게 지급되는 보상
- **반응형_레이아웃**: 데스크톱, 태블릿, 모바일 등 다양한 화면 크기에 맞게 자동 조정되는 UI 레이아웃

## 요구사항

### 요구사항 1: 사용자 인증

**사용자 스토리:** 사용자로서, 이메일과 비밀번호로 로그인하고 싶다. 그래야 커뮤니티의 기능을 안전하게 이용할 수 있다.

#### 인수 조건

1. WHEN 사용자가 유효한 이메일과 비밀번호를 입력하고 로그인 버튼을 클릭하면, THE 인증_모듈 SHALL 사용자를 인증하고 대시보드로 이동시킨다.
2. WHEN 사용자가 유효하지 않은 이메일 또는 비밀번호를 입력하고 로그인 버튼을 클릭하면, THE 인증_모듈 SHALL "이메일 또는 비밀번호가 올바르지 않습니다" 오류 메시지를 표시한다.
3. WHEN 사용자가 로그아웃 버튼을 클릭하면, THE 인증_모듈 SHALL 세션을 종료하고 로그인 페이지로 이동시킨다.
4. IF 사용자의 세션이 만료되면, THEN THE 인증_모듈 SHALL 사용자를 로그인 페이지로 이동시키고 "세션이 만료되었습니다. 다시 로그인해주세요" 메시지를 표시한다.
5. WHEN 사용자가 회원가입 양식에 이메일, 비밀번호, 닉네임을 입력하고 가입 버튼을 클릭하면, THE 인증_모듈 SHALL 새 계정을 생성하고 로그인 페이지로 이동시킨다.
6. IF 사용자가 이미 등록된 이메일로 회원가입을 시도하면, THEN THE 인증_모듈 SHALL "이미 등록된 이메일입니다" 오류 메시지를 표시한다.

### 요구사항 2: 대시보드

**사용자 스토리:** 로그인한 사용자로서, 대시보드에서 커뮤니티의 주요 활동을 한눈에 확인하고 싶다. 그래야 빠르게 관심 있는 콘텐츠에 접근할 수 있다.

#### 인수 조건

1. WHEN 사용자가 로그인에 성공하면, THE 대시보드 SHALL 사용자의 닉네임과 함께 환영 메시지를 표시한다.
2. THE 대시보드 SHALL 최근 게시판 게시글 목록을 최대 5개까지 표시한다.
3. THE 대시보드 SHALL 이번 달 꿀팁 랭킹 상위 3개 게시글을 표시한다.
4. THE 대시보드 SHALL 사용자 본인의 이번 달 꿀팁 게시글 좋아요 수와 현재 순위를 표시한다.
5. WHEN 사용자가 대시보드의 게시글 항목을 클릭하면, THE 웹앱 SHALL 해당 게시글 상세 페이지로 이동시킨다.

### 요구사항 3: 게시판

**사용자 스토리:** 사용자로서, 게시판에서 다른 부모들과 육아 정보를 공유하고 소통하고 싶다. 그래야 육아에 도움이 되는 정보를 얻을 수 있다.

#### 인수 조건

1. THE 게시판 SHALL 게시글 목록을 최신순으로 정렬하여 페이지당 10개씩 표시한다.
2. WHEN 사용자가 제목과 본문을 입력하고 작성 버튼을 클릭하면, THE 게시판 SHALL 새 게시글을 저장하고 게시글 목록에 표시한다.
3. WHEN 사용자가 본인이 작성한 게시글의 수정 버튼을 클릭하면, THE 게시판 SHALL 제목과 본문을 수정할 수 있는 편집 화면을 표시한다.
4. WHEN 사용자가 본인이 작성한 게시글의 삭제 버튼을 클릭하면, THE 게시판 SHALL 삭제 확인 대화상자를 표시하고, 확인 시 해당 게시글을 삭제한다.
5. WHEN 사용자가 게시글 목록에서 특정 게시글을 클릭하면, THE 게시판 SHALL 해당 게시글의 제목, 본문, 작성자 닉네임, 작성일시, 좋아요 수를 표시한다.
6. WHEN 사용자가 게시글의 좋아요 버튼을 클릭하면, THE 게시판 SHALL 해당 게시글의 좋아요 수를 1 증가시키고 버튼 상태를 활성화로 변경한다.
7. WHEN 사용자가 이미 좋아요를 누른 게시글의 좋아요 버튼을 다시 클릭하면, THE 게시판 SHALL 해당 게시글의 좋아요 수를 1 감소시키고 버튼 상태를 비활성화로 변경한다.
8. IF 사용자가 제목 또는 본문을 비워둔 채 작성 버튼을 클릭하면, THEN THE 게시판 SHALL "제목과 본문을 모두 입력해주세요" 오류 메시지를 표시한다.
9. WHEN 사용자가 검색어를 입력하고 검색 버튼을 클릭하면, THE 게시판 SHALL 제목 또는 본문에 검색어가 포함된 게시글 목록을 표시한다.

### 요구사항 4: 월별 꿀팁 랭킹 시스템

**사용자 스토리:** 사용자로서, 매월 육아 꿀팁을 공유하고 좋아요 기반 랭킹에 참여하고 싶다. 그래야 유용한 정보를 공유하는 동기를 얻고 리워드를 받을 수 있다.

#### 인수 조건

1. WHEN 사용자가 꿀팁 게시글 작성 시 "꿀팁 이벤트 참여" 옵션을 선택하면, THE 랭킹_시스템 SHALL 해당 게시글을 이번 달 꿀팁 랭킹 대상으로 등록한다.
2. THE 랭킹_시스템 SHALL 해당 월에 등록된 꿀팁_게시글을 좋아요 수 기준 내림차순으로 정렬하여 랭킹 페이지에 표시한다.
3. WHEN 동일한 좋아요 수를 가진 꿀팁_게시글이 존재하면, THE 랭킹_시스템 SHALL 먼저 작성된 게시글에 높은 순위를 부여한다.
4. WHEN 새로운 달이 시작되면, THE 랭킹_시스템 SHALL 이전 달의 랭킹을 확정하고 새로운 달의 랭킹을 초기화한다.
5. THE 랭킹_시스템 SHALL 이전 달의 확정된 랭킹 결과를 아카이브 페이지에서 조회할 수 있도록 보관한다.
6. WHEN 월별 랭킹이 확정되면, THE 랭킹_시스템 SHALL 상위 3위까지의 사용자에게 리워드를 지급한다.
7. THE 웹앱 SHALL 사용자의 리워드 내역을 마이페이지에서 조회할 수 있도록 표시한다.
8. WHILE 해당 월의 랭킹이 진행 중인 동안, THE 랭킹_시스템 SHALL 실시간으로 좋아요 수 변동에 따라 랭킹 순위를 갱신한다.

### 요구사항 5: 반응형 레이아웃

**사용자 스토리:** 사용자로서, 데스크톱, 태블릿, 모바일 등 다양한 기기에서 편리하게 웹앱을 이용하고 싶다. 그래야 언제 어디서든 커뮤니티에 참여할 수 있다.

#### 인수 조건

1. WHILE 화면 너비가 768px 미만인 경우, THE 웹앱 SHALL 모바일 레이아웃으로 단일 컬럼 구조를 표시한다.
2. WHILE 화면 너비가 768px 이상 1024px 미만인 경우, THE 웹앱 SHALL 태블릿 레이아웃으로 2컬럼 구조를 표시한다.
3. WHILE 화면 너비가 1024px 이상인 경우, THE 웹앱 SHALL 데스크톱 레이아웃으로 전체 너비 구조를 표시한다.
4. THE 웹앱 SHALL 모든 인터랙티브 요소(버튼, 입력 필드, 링크)의 터치 영역을 최소 44x44px로 유지한다.
5. WHILE 모바일 레이아웃이 활성화된 경우, THE 웹앱 SHALL 내비게이션 메뉴를 햄버거 메뉴 아이콘으로 축소하여 표시한다.
6. THE 웹앱 SHALL 모든 페이지에서 텍스트, 이미지, 버튼 등의 UI 요소가 화면 크기에 따라 비율을 유지하며 조정되도록 한다.
